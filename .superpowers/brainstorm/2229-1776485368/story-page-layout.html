Story flows top-to-bottom in sections: Problem → Architecture → Try It → Results. Customer scrolls naturally. Simple, familiar. But the architecture diagram and interactive area can't be seen at the same time.

B: Split View — Visual Left, Interactive Right

Left panel shows the visual story (animated diagrams, before/after comparisons). Right panel is the interactive area (inputs, results, live data). Both visible at all times. The visual updates as you interact — architecture nodes glow when data flows through them.

C: Stepped Wizard — One Thing at a Time

Each story is a 3-5 step wizard with progress bar. One full-width visual per step with narrative text below. Next/Back navigation. Maximum visual impact per step — no split attention. Like the Recovery page's step flow, but for every feature.
