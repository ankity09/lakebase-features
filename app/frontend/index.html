Overview

Lakebase is Databricks' Postgres-compatible operational database built on top of the Lakehouse. It combines the familiarity of PostgreSQL with Lakehouse-native features: automatic zero-ETL sync to Delta Lake, database branching for safe schema migrations, autoscaling compute, scale-to-zero for cost savings, and built-in pgvector support for AI workloads. This demo uses AppShield Analytics — a fictitious cybersecurity SaaS company — to showcase these capabilities with realistic telemetry, feature store, and ML prediction data.

Branching creates instant, zero-copy database copies — like git branches for your database. Use branches to safely test schema migrations, onboard new product groups, or validate performance changes before merging to production. Each branch has its own compute endpoint with no storage duplication.

Autoscaling

Autoscaling automatically adjusts compute resources (measured in Compute Units / CUs) based on your workload. Set a minimum and maximum CU range, and Lakebase dynamically scales within those bounds. Each CU provides approximately 4 GiB of memory. Scale up for heavy analytical queries, scale down during quiet periods.

Scale to Zero

Scale to Zero suspends compute when the database is idle — you only pay for storage. When a new connection arrives, Lakebase automatically wakes up (typically in 1-3 seconds). This is ideal for dev/staging environments, periodic batch workloads, or cost-sensitive applications.
⚠ Warning: Suspending will temporarily disconnect this app from Lakebase. The app will show errors until the endpoint resumes (~5-30 seconds). Only demo this when you're prepared for a brief interruption.

Read Replicas

Read Replicas provide dedicated read-only endpoints that offload analytical queries from your primary R/W endpoint. Route heavy reporting queries to replicas while keeping OLTP writes fast on the primary. Replicas share storage and receive changes in near-real-time.

Feature Store

Lakebase as Online Feature Store: Pre-computed ML features stored in Lakebase are served with single-digit millisecond latency for real-time inference. Compare this to reading features from Delta Lake (batch, hundreds of ms). AppShield stores customer behavioral features (request counts, geo diversity, cookie patterns) for its real-time app classification model.

pgvector Search

pgvector is the Postgres extension for vector similarity search. Lakebase supports pgvector natively, enabling you to store embeddings alongside relational data and perform similarity queries using cosine distance, L2 distance, or inner product. AppShield stores security event embeddings (384-dimensional) for semantic search across threat intelligence data.

AI Memory

Lakebase as persistent memory for AI agents. Conversations, facts, and knowledge are stored in Postgres with pgvector for semantic retrieval. When the agent receives a new message, it queries Lakebase for related memories using embedding similarity — enabling context-aware, personalized responses across sessions. This pattern powers long-term memory for chatbots, copilots, and autonomous agents.
